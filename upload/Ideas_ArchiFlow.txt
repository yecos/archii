IDEA 1: IMPLEMENTACION DE PROYECTO CON ALERTAS Y AJUSTE DE CRONOGRAMA
================================================================================

DESCRIPCION GENERAL:
Desarrollar un sistema inteligente dentro de la aplicacion que permita gestionar
el cronograma de proyectos de construccion con capacidad de detectar desviaciones,
generar alertas automaticas y reajustar las fechas del cronograma de forma
automatica o asistida cuando se presenten retrasos o cambios.

FUNCIONALIDADES PRINCIPALES:

1. Sistema de Alertas Automaticas
   - Deteccion automatica de tareas atrasadas (fecha limite vencida)
   - Alertas preventivas cuando una tarea esta proxima a vencer (ej. 3 dias)
   - Alertas de dependencias bloqueadas (tareas que no pueden iniciar porque
     su tarea predecesora no ha sido completada)
   - Notificaciones en tiempo real con indicadores visuales (colores por
     nivel de urgencia: rojo para urgente, amarillo para advertencia,
     azul para informativo)
   - Contador de alertas no leidas en la barra superior
   - Panel lateral de alertas en la seccion de implementacion
   - Posibilidad de marcar alertas como leidas individualmente o todas

2. Reajuste Automatico del Cronograma
   - Cuando una tarea se atrasa, el sistema puede reajustar las fechas de
     las tareas dependientes de forma automatica
   - Dos modos de reajuste:
     * "Mover todas las fechas": Desplaza todas las tareas posteriores
       en N dias
     * "Mover solo dependientes": Desplaza unicamente las tareas que
       dependen directamente de la tarea atrasada
   - Selector de cantidad de dias a desplazar (slider + input numerico)
   - Vista previa de tareas afectadas antes de confirmar el reajuste
   - Registro historico de cambios al cronograma (quien, cuando, por que)

3. Visualizacion del Impacto en la Linea de Tiempo
   - Vista de timeline/Gantt que muestra todas las tareas con sus fechas
     de inicio y fin
   - Colores por estado de la tarea:
     * Verde: Completado o dentro del plazo
     * Amarillo: Proximo a vencer (3 dias o menos)
     * Rojo: Atrasado
   - Agrupacion de tareas por proyecto
   - Navegacion por rango de fechas
   - Click en tarea para ver detalle completo (responsable, prioridad,
     dependencias, fechas, enlace al acta de reunion si aplica)

4. Tablero Kanban
   - Cinco columnas: Por iniciar, En progreso, En revision, Completado,
     Atrasado
   - Arrastrar y soltar tareas entre columnas para cambiar estado
   - Indicadores visuales por prioridad (Alta, Media, Baja)
   - Badge que indica si la tarea fue generada desde un acta de reunion
   - Filtro por proyecto
   - Conteo de tareas por columna

5. Gestion de Tareas
   - Crear, editar y eliminar tareas
   - Campos: titulo, descripcion, proyecto, responsable, prioridad, estado,
     fecha de inicio, fecha de fin, dependencias
   - Asociar tareas a proyectos existentes
   - Vincular dependencias entre tareas del mismo proyecto
   - Marcar tareas como "Atrasado" automaticamente cuando la fecha limite
     ha pasado y no estan completadas

IDEA 2: BITACORA DE OBRA
================================================================================

DESCRIPCION GENERAL:
Crear una seccion dedicada al registro diario de actividades en obra, que
permita documentar de forma completa y organizada todo lo que ocurre en
la jornada de trabajo, incluyendo clima, personal, equipos, avances y
observaciones.

FUNCIONALIDADES PRINCIPALES:

1. Registro Diario de Actividades
   - Formulario completo para crear entradas de bitacora con:
     * Fecha del registro
     * Proyecto asociado
     * Actividades realizadas (texto descriptivo detallado)
     * Observaciones adicionales
   - Edicion de entradas existentes
   - Eliminacion de entradas

2. Registro Climatologico
   - Seleccion de clima del dia con iconos:
     * Soleado, Nublado, Lluvioso, Parcialmente nublado, Tormenta
   - Registro de temperatura en grados Celsius

3. Control de Avance
   - Registro de avance real del proyecto (porcentaje con slider)
   - Registro de avance planificado (porcentaje con slider)
   - Comparativa visual real vs. planificado
   - Indicador de estado: "Al dia", "X% adelante", "X% atras"
   - Grafico de barras comparativo (real vs. planificado)

4. Galeria de Fotos
   - Adjuntar fotos con descripcion/caption
   - Vista de galeria en grilla (2 o 3 columnas)
   - Lightbox para ver fotos en pantalla completa
   - Navegacion entre fotos (anterior/siguiente)
   - Posibilidad de eliminar fotos

5. Personal en Obra
   - Registro de cada persona que trabajo en la jornada:
     * Nombre, Rol, Horas trabajadas
   - Lista editable (agregar/eliminar personal)
   - Vista resumen con avatar, nombre, rol y horas

6. Equipos Utilizados
   - Registro de equipos y maquinaria usada:
     * Nombre del equipo, Horas de uso
   - Lista editable (agregar/eliminar equipos)
   - Vista resumen con icono, nombre y horas

7. Firmas Digitales
   - Seccion de firmas para roles especificos:
     * Supervisor de obra
     * Interventor
     * Residente
   - Registro de nombre y hora de firma
   - Cada rol solo puede firmar una vez por entrada

8. Lista y Navegacion de Entradas
   - Vista de lista con tarjetas por cada entrada
   - Cada tarjeta muestra: fecha, proyecto, clima, temperatura,
     resumen de actividades, avance, cantidad de personal, equipos y fotos
   - Filtro por proyecto
   - Busqueda por texto (actividades, observaciones)
   - Navegacion por mes
   - Click en tarjeta para ver detalle completo

9. Vista de Detalle
   - Muestra toda la informacion de la entrada en formato profesional
   - Cabecera con fecha completa y nombre del proyecto
   - Secciones organizadas: clima, avance, actividades, observaciones,
     personal, equipos, fotos, firmas
   - Boton de editar

IDEA 3: ACTAS DE REUNION CON GENERACION AUTOMATICA DE TAREAS
================================================================================

DESCRIPCION GENERAL:
Desarrollar un sistema para crear actas de reunion de obra completas, que
incluya participantes, agenda, discusiones, acuerdos y compromisos, y que
a partir de los compromisos registrados genere automaticamente tareas en
el modulo de implementacion con responsable, fecha y descripcion asignados.

FUNCIONALIDADES PRINCIPALES:

1. Creacion de Actas de Reunion
   - Formulario completo para crear reuniones con:
     * Titulo de la reunion
     * Fecha, hora y lugar
     * Tipo de reunion (Reunion de obra, Comite tecnico,
       Reunion con cliente, Otra)
     * Proyecto asociado
     * Estado (Programada, Realizada, Cancelada)

2. Gestion de Participantes
   - Agregar participantes con nombre y rol:
     * Asistente, Moderador, Secretario
   - Vista de participantes con avatar (iniciales) y badge de rol
   - Posibilidad de eliminar participantes

3. Agenda y Puntos de Discusion
   - Crear puntos de agenda numerados
   - Para reuniones marcadas como "Realizada", se habilita la seccion
     de discusiones donde se pueden registrar notas por cada punto
     de agenda
   - Eliminar puntos de agenda

4. Registro de Acuerdos
   - Lista de acuerdos alcanzados en la reunion
   - Cada acuerdo se muestra con un check verde
   - Agregar y eliminar acuerdos

5. Registro de Compromisos (CLAVE)
   - Cada compromiso incluye:
     * Descripcion del compromiso (que se va a hacer)
     * Responsable (quien lo va a hacer)
     * Fecha de entrega (cuando se debe entregar)
     * Prioridad (Alta, Media, Baja)
   - Vista destacada con fondo dorado para diferenciar de acuerdos
   - Agregar y eliminar compromisos

6. Generacion Automatica de Tareas desde el Acta
   - Cuando una reunion esta marcada como "Realizada" y tiene compromisos,
     aparece un boton destacado para "Generar Tareas"
   - Al hacer click, se abre un dialogo que muestra:
     * Informacion de la reunion (titulo, fecha)
     * Vista previa de las tareas que se van a crear (una por compromiso)
     * Para cada tarea se muestra: descripcion, responsable, fecha,
       prioridad
   - Animacion de carga durante la generacion
   - Una vez generadas, las tareas aparecen automaticamente en el
     modulo de Implementacion con:
     * Titulo = descripcion del compromiso
     * Descripcion = referencia al acta de reunion
     * Proyecto = proyecto de la reunion
     * Responsable = responsable del compromiso
     * Prioridad = prioridad del compromiso
     * Estado = "Por iniciar"
     * Fecha de inicio = fecha actual
     * Fecha de fin = fecha de entrega del compromiso
     * Vinculo al acta de reunion origen (badge "Acta")
   - Estado de exito con opcion de navegar a las tareas creadas

7. Vista de Detalle del Acta
   - Muestra toda la informacion de la reunion en formato profesional
   - Titulo con badges de estado y tipo
   - Detalles: fecha, hora, lugar
   - Lista de participantes con roles
   - Agenda numerada
   - Discusiones por punto de agenda
   - Acuerdos con check
   - Compromisos con indicador de si ya se genero la tarea asociada
   - Panel lateral con:
     * Boton para generar tareas (si aplica)
     * Lista de tareas ya generadas
     * Resumen estadistico (participantes, agenda, acuerdos, compromisos)
   - Boton para marcar reunion como "Realizada"
   - Botones de editar y eliminar

8. Lista y Navegacion de Reuniones
   - Vista de lista con tarjetas por cada reunion
   - Cada tarjeta muestra: fecha, titulo, tipo, estado, participantes,
     ubicacion, hora
   - Filtro por proyecto
   - Busqueda por titulo
   - Click en tarjeta para ver detalle completo

INTEGRACION CON MICROSOFT PROJECT
================================================================================

DESCRIPCION GENERAL:
Posibilidad de vincular archivos de Microsoft Project (.mpp) con ArchiFlow
para importar cronogramas existentes y exportar datos de la aplicacion.

OPCIONES DISPONIBLES:

1. Import/Export CSV/Excel (Recomendado para empezar)
   - Exportar desde MS Project a CSV/Excel
   - Subir el archivo CSV a ArchiFlow
   - ArchiFlow importa automaticamente tareas, fechas, responsables y
     dependencias
   - Tambien se puede exportar desde ArchiFlow a CSV para abrir en
     MS Project
   - Ventajas: Facil de implementar, sin licencias adicionales
   - Desventajas: Proceso manual, no en tiempo real

2. MPXJ - Lectura directa de archivos .mpp
   - Subir archivo .mpp directamente a ArchiFlow
   - La aplicacion lo lee sin necesidad de exportar antes
   - Permite importar: tareas, recursos, costos, calendarios laborales,
     hitos, dependencias
   - Ventajas: Lectura directa, mas datos, no requiere MS Project
   - Desventajas: Mas complejo de implementar
   - Costo: Gratis (Open Source)

3. Microsoft Graph API (En tiempo real)
   - Conexion directa con MS Project Online / Project Server
   - Sincronizacion automatica de datos
   - Ideal para empresas con licencias Microsoft 365
   - Ventajas: Tiempo real, trabajo en equipo simultaneo
   - Desventajas: Requiere suscripcion a MS Project Online, configuracion
     OAuth/Azure AD
   - Costo: Requiere licencia de MS Project Online

RECOMENDACION:
Empezar con CSV/Excel + MPXJ combinados para tener flexibilidad completa.
Luego agregar Microsoft Graph API si el equipo crece y usa MS Project Online.


================================================================================
TECNOLOGIAS ACTUALES DE ARCHIFLOW
================================================================================

- Framework: Next.js 16 con App Router
- Lenguaje: TypeScript
- Estilos: Tailwind CSS 4 + shadcn/ui
- Estado: Zustand (con persistencia en localStorage)
- Graficos: Recharts
- Animaciones: Framer Motion
- Fechas: date-fns (locale espanol)
- Base de datos: Prisma + SQLite
- Tema: Dark mode con acentos dorados (gold)
- Tipografia: DM Sans + DM Serif Display
- Iconos: Lucide React
