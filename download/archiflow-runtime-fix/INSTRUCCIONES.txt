========================================
ARCHIFLOW - FIX "This page couldn't load"
========================================

PROBLEMA IDENTIFICADO:
La app compila correctamente en Vercel pero falla en el navegador con 
"This page couldn't load". Se identificaron 2 causas principales:

1. firebase.ts — Este archivo importa el SDK MODULAR de Firebase 
   (firebase/app, firebase/auth, etc.) y ejecuta initializeApp() 
   al nivel del módulo. Turbopack puede incluirlo accidentalmente
   en el bundle del cliente, causando conflicto con el SDK compat
   cargado via <script> en layout.tsx.

2. firebase-service.ts línea 10 — Tiene:
   import type { User as FirebaseUser } from 'firebase/auth';
   Aunque debería borrarse por TypeScript, Turbopack puede NO 
   borrarla correctamente, haciendo que el paquete 'firebase' npm 
   se incluya en el bundle del cliente.

INSTRUCCIONES:
=============

1. ELIMINAR firebase.ts:
   Elimina o renombra el archivo:
   src/lib/firebase.ts
   
   En tu terminal:
   rm src/lib/firebase.ts
   (o renómbralo a firebase.ts.bak)

2. REEMPLAZAR page.tsx:
   Copia el archivo page.tsx de esta carpeta a:
   src/app/page.tsx

3. REEMPLAZAR firebase-service.ts:
   Copia el archivo firebase-service.ts de esta carpeta a:
   src/lib/firebase-service.ts

4. VERIFICAR que HomeContent.tsx existe:
   Asegúrate de que src/app/HomeContent.tsx existe.
   Si no existe, tu page.tsx original (el grande con todo el código)
   debe renombrarse a HomeContent.tsx.

5. Commit y push:
   git add .
   git commit -m "fix: runtime crash - remove firebase modular SDK imports"
   git push

NOTA IMPORTANTE:
- page.tsx ahora es un wrapper mínimo que carga HomeContent.tsx
  dinámicamente con useEffect + import(). No tiene NINGUNA 
  importación de Firebase.
- firebase-service.ts ya NO importa de 'firebase/auth'. 
  Todos los tipos se definen localmente.
- firebase.ts fue eliminado porque no se usa en ningún componente
  del cliente y solo causaba problemas.
